Intelligence Design System

This is the shared component library for the Intelligence platform. Every component here is a reusable template — not tied to any specific tool. Tools compose these components to build their interfaces. When building a new tool or page, draw from these patterns; don't invent new ones unless nothing here fits. Sample content is illustrative only.

Usage principles
  • Use CSS variables for all colors, fonts, radii, and borders — never hardcode values.
  • Components are composable: section cards wrap prose, item lists, stat grids, timelines, or any other content type.
  • Nothing below 13px rendered font size. Body text is 15–16px.
  • Dark background throughout. No light/white backgrounds anywhere.
  • Must work beautifully on desktop, tablet, and phone. Phone is not a degraded experience.
Accent color semantics
  • Warm — primary actions, active states, attention, the brand accent.
  • Sage — positive states, success, healthy, compliant.
  • Rose — negative states, errors, destructive actions, warnings.
  • Blue — informational, neutral-active, supplementary data.
  • Lavender — categorization, tertiary grouping, supplementary.
